# ClassicWebAgent 端到端测试用例与实验报告

> **项目**: ClassicWebAgent — 自然语言驱动的网页多模态 Agent
> **架构**: Director(LLM) + SubAgent(VLM) 双层编排
> **本文档**: 测试用例定义 + 实验结果记录

---

## 一、测试方法论

### 1.1 用例设计原则

每个测试用例必须同时满足：
- **真实需求**：一个普通人确实会为此打开浏览器
- **必须用浏览器**：LLM 训练数据答不出（实时/动态/当前信息）
- **系统可执行**：当前系统支持导航、点击、输入、提取文本、滚动、标签页管理

### 1.2 评估指标

| 维度 | 权重 | 说明 | 数据来源 |
|:----|:---:|------|---------|
| 任务完成度 | 0.5 | 报告覆盖了多少用户要求的内容 | LLM 按 eval_criteria 评分 |
| 信息质量 | 0.3 | 数据准确性/具体性/来源可信度 | LLM 评分 |
| 子任务完成率 | 0.2 | 浏览操作的成功比例 | sub_tasks.json |

**总分公式**：`总分 = 完成(0/1) × (任务完成度×0.5 + 信息质量×0.3 + 子任务完成率×0.2)`

### 1.3 运行与评估命令

```bash
# 运行所有测例
python scripts/run_test_cases.py

# 运行指定测例
python scripts/run_test_cases.py --id 1 3 5

# 评估结果
python scripts/evaluate.py --range [起始编号-结束编号]

# 仅统计不调 LLM
python scripts/evaluate.py --range [起始编号-结束编号] --no-llm
```

### 明确型（指令详细、步骤明确）

#### TC-01: 今天北京天气怎么样

**任务描述**：
```
帮我查一下今天北京的天气，我需要知道：
- 当前温度和全天温度范围（最高温、最低温）
- 降水概率和风力大小
- 适合户外活动吗
```

| 属性 | 值 |
|------|------|
| **场景** | 天气查询 |
| **难度** | ⭐ |
| **LLM 参与** | 不需要 |
| **预期步骤** | 导航到天气网站 → 提取天气信息 |
| **确定性** | 确定性，明确的指令链 |

**评分标准**：
- 是否包含当前温度及温度范围（0-30分）
- 是否包含降水概率（0-25分）
- 是否包含风力信息（0-25分）
- 是否给出户外活动建议（0-20分）
